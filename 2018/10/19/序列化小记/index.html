#序列化小记#

##序列化与反序列化##
将对象转换为字节序列的过程称为对象的序列化；反之字节序列恢复为对象的过程称之为反序列化。

##序列化主要有两种用途##

  1. 把对象的字节序列永久地保存到硬盘上，如保存到在文件中或数据库中；
  2. 在网络上传送对象的字节序列，传给另一个模块或系统；

##序列化的方法##

  1. java原生的序列化
    Java原生序列化方法即通过Java原生流(InputStream和OutputStream之间的转化)的方式进行转化。需要注意的是JavaBean实体类必须实现Serializable接口，否则无法序列化。
    优点：不需要引入第三方包
    缺点：效率比较低
  2. json序列化
    Json序列化一般会使用jackson包，通过ObjectMapper类来进行一些操作，比如将对象转化为byte数组或者将json串转化为对象。现在的大多数公司都将json作为服务器端返回的数据格式。比如调用一个服务器接口，通常的请求为xxx.json?a=xxx&b=xxx的形式
    优点：直观，能和js配对使用
    缺点：效率低
  3. xml序列化
    XML 序列化只将对象的公共字段和属性值序列化为 XML 流。 XML 序列化不包括类型信息，不能转换方法、索引器、私有字段或只读属性（只读集合除外。 例如，如果 Library 命名空间中存在 Man 对象，则不能保证将它反序列化为同一类型的对象。
    优点：直观、不受环境限制，发送端时java ,接收端可以时任何js,python
    缺点：标签太多，效率低
  4. FastJson序列化
    fastjson 是由阿里巴巴开发的一个性能很好的Java 语言实现的 Json解析器和生成器。
    优点：速度快；能强大，完全支持java bean、集合、Map、日期、Enum，支持范型和自省。
    缺点：使用时候需引入FastJson第三方jar包
  5. ProtoBuff序列化
    Protostuff是基于Google protobuff技术的Java版本，直接使用原生的protobuff是需要数据结构的预编译过程，需要编写.proto格式的配置文件，再通过protobuf提供的工具翻译成目标语言代码，而Protostuff动态支持了protobuff的预编译的过程。
    优点：跨语言；数据压缩率大
    缺点：压缩后数据二进制的方式存储，无法直接读取编辑；需要自己写.proto文件，通过编译器将其转换为java文件（百度做过升级，
    通过对JavaBean中的字段进行注解）

##总结##
根据序列化的性能、用途等条件选择合适的序列化方式；如偏向于网络传输的可选择选择压缩率较高的方式，若是序列化是为了持久化的可选择xml/json模式持久化后的数据比较直观。
